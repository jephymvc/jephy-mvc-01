# Overview

{{ site.title }} is a lightweight, modular PHP MVC framework built for developers who want clean architecture, high performance, and full control over their web applications. It follows the Model-View-Controller pattern to separate business logic, presentation, and data handling—making your code more maintainable, scalable, and testable.

Template Engine: Unlike many lightweight frameworks that rely on raw PHP in views, {{ site.title }} natively integrates the Smarty Engine—a powerful, feature-rich template system that keeps your presentation layer clean, secure, and easy to manage.

Key Features

  • Modular MVC Structure – Clear separation of concerns with Models, Views, and Controllers.
  • Smarty Template Engine – Write clean, logic-free templates with caching, layouts, custom functions, and built-in security against XSS.
  • Simple Routing – Define clean, RESTful routes with support for dynamic parameters and middleware.
  • Lightweight & Fast – Minimal overhead with no unnecessary dependencies.
  • Database Abstraction – Built-in query builder and active record pattern (or ORM).
  • Middleware Support – Filter HTTP requests before they reach your controllers.
  • Request & Response Handling – Object-oriented wrappers for input, session, cookies, and output.
  • CSRF & XSS Protection – Built-in security helpers (Smarty auto-escaping adds an extra layer).
  • Error Handling & Debugging – Custom error pages and debug mode with detailed logs.

# Getting Started

System Requirements - Option 1

  • PHP 8.2+
  • XAMPP

System Requirements - Option 2

  • PHP 8.2+
  • Apache/Nginx
  • MySQL/MariaDB, Postgres or SQLite

# Installation

To install JephyPHP, visit https://github.com/jephymvc/jephy-mvc-01.git and clone the repository or download the zip file.

Working with Windows

If you are working with option 1, extract the zip file into the C:/xampp/htdocs path. That is the contents of /src directory.

# Configuration

Open the config.conf file located in src/jephy-mvc/ folder and configure based on your system setup. You can see the table below for the explanation of each configurable fields:

Field Description
site.app_debug Allows to enable the display of debug data. It is set to true by default. You will need to set to false on the production server.
site.version This states the current version of your application.
site.name This specifies the name of your website or project.
site.email Optional - this specifies the contact email that is available for the public to reach when on your website.
site.phone Optional - this specifies the contact phone that is available for the public to reach when on your website.
site.home The URL of your website.
site.app_path This specifies the absolute path to your App\ namespace. It is usually {your_absolute_path}/jephy-mvc/app. You only need to update according to your setup.
site.encryption_key Login session data are encrypted using AES algorithm for security purpose. So, you only need to come up a difficult to guess set of characters and It is advised to change regularly.
site.jwt_secret_key Tokenized session data are encrypted using JWT algorithm for security purpose. So, you only need to come up a difficult to guess set of characters and It is advised to change regularly.
site.jwt_algo This is the JSON Web Token algorithm used in encryption. It usually defaults to HS256.
site.jwt_ttl This is the time it takes JSON Web Tokens to get expire.
site.address Company location address.
site.title Default title to your home page. It is for SEO needs.
site.keywords Suitable keywords for your business in case it is being searched on Search engines. Good for SEO.
site.description Suitable description for your business in case it is being searched on Search engines. Good for SEO.
site.url.home It is alternative to site.home.
site.url.third_party_platform.github Optional.
site.author.fullname Optional - usually for the lead or developer in charge.
site.author.email Optional - usually for the lead or developer in charge.
database.driver This specifies the database driver the application is working. You have option between mysql, pgsql and sqlite for MySQL/MariaDB, PostgresDB and SQLite respectively.
